# Padrão: Framework Numerado

## Estrutura

```
[Afirmação que o resultado não é aleatório]. É [sistema/método/processo].

[Contextualização pessoal: após X empresas/anos/situações...]

[NOME DO FRAMEWORK EM MAIÚSCULAS]:

1. [PILAR 1]
→ [detalhe 1]
→ [detalhe 2]

2. [PILAR 2]
→ [detalhe 1]
→ [detalhe 2]

[...]

Com [isso/esses dados/esse framework] você consegue:
→ [benefício 1]
→ [benefício 2]
→ [benefício 3]

[Prazo ou condição] para implementar. [Ressalva humanizadora].

[CTA de salvamento ou pergunta]

#hashtag1 #hashtag2 #hashtag3
```

## Por que funciona

Frameworks são salvos e compartilhados. A estrutura numerada é fácil de consumir no scroll. A promessa de resultado concreto ("você consegue...") justifica o engajamento.

## Exemplos da Sprint 01

**Post #12 — Os 4 Pilares do Pipeline Previsível**
> "Pipeline previsível não é sorte. É sistema. [...] OS 4 PILARES DO PIPELINE PREVISÍVEL: 1. VOLUME [...] 2. VELOCIDADE [...] Salva esse post."

## Variações

- "Os X passos para..."
- "X erros que [resultado]"
- "X perguntas que todo [cargo] deveria responder"
- "O framework de X etapas para..."

## Boas práticas

- Máximo 5 itens (acima disso perde escaneabilidade)
- Cada pilar em MAIÚSCULAS + bullets com → 
- Terminar com "Salva esse post" — sinaliza valor de referência
- A contextualização pessoal no início dá credibilidade ao framework

## Quando usar

- Quando você tem um sistema que funcionou e pode ser replicado
- Temas de processo, operação, metodologia
- Quando quer gerar salvamentos (métrica de valor no LinkedIn)
